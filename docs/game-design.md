# Macrocosmo — Game Design Document

## コンセプト

Stellarisライクな宇宙4Xストラテジー。最大の特徴は**光速の壁**をゲームメカニクスの中心に据えている点。通信・情報・命令すべてに距離に応じた遅延が発生し、プレイヤーは不確実性の中で委任と計画による意思決定を強いられる。

## 確定事項

### 光速制約と通信

- デフォルトでは通信は光速を超えられない
- あらゆる命令はプレイヤーの現在位置からの距離に応じて実行までにタイムラグが発生する
- プレイヤーが知覚できる情報も距離に応じて古くなる（遠方の星系ほど過去の状態しか見えない）

### FTL航法

- FTLは**弾道飛行のみ**（出発時に目的地を決定、途中変更不可）
- ハイパーレーンは存在しない。最大到達距離の範囲内にある**任意の既知星系**に飛行可能
- FTL中は**観測・通信ともに不可能**
- 領有済み星系2つの間に**港湾施設**を整備することでFTL到達時間を短縮できる
  - 港湾は出発側の**加速補助**として機能する（減速補助はない）
  - 目的地の港湾が移動中に破壊された場合、到着時に**損傷を受ける**が飛び越えはしない

### プレイヤーの位置と移動

- プレイヤーは物理的な位置を持ち、**現在位置でのみ即座に命令を実行**できる
- 現在位置にある艦船を指定して搭乗し、移動できる
- 搭乗中の艦船がいる星系の入植済み惑星を指定して、その惑星を現在位置にできる
- 移動手段の選択：
  - **FTL移動** — 高速だが移動中は完全ブラックアウト（通信・管理不能）
  - **亜光速移動** — 低速だが移動中も（遅延付きで）通信可能
- 通信艦を含む艦隊を編成して移動することが推奨される（単独移動時は警告を表示）

### 亜光速飛行

- 初期段階で**光速の75%**程度
- テクノロジーによりアップグレード可能

### 通信手段の段階

1. **通信艦**（手動） — 低コスト。プレイヤーが手動で命令・経路を設定して飛ばす
2. **定時連絡網**（自動） — 通信艦を自動化し、定期的に2点間で情報を往復させる
3. **即時通信インフラ**（エンドコンテンツ） — 2点間で即時通信を実現する高コスト施設

### 自律AI（委任システム）

- 遠隔地の艦隊・総督に対して交戦規定や条件付き命令を事前設定できる
- 設定変更の反映にはプレイヤー位置からの**通信遅延**がかかる
- AIの性能は**テクノロジーツリー**でアップグレード可能
- AIに**性格特性**を持たせる可能性あり

### 測量と探索

- 領有可能な星系のおおよその位置は判明しているが、正確な測量には一定距離まで接近が必要
- 初期段階の測量範囲は**約5光年**
- 未測量星系にはFTLで直接飛行できない（亜光速で接近→測量→以後FTL可能）
- **星間ガス密度が高い方向**にある星系は視認できない

### 早期警戒

- **受動的探知** — 天文台等による光学的・電磁的監視
- **能動的探知** — 哨戒船の定期巡回、前方監視ステーション
- FTL航行には検出可能な**シグネチャ**が発生するが、初期テクノロジーでは検出不可能

### 研究

- 研究は**首都星系に集約**される
- 遠隔地で発生した研究ポイントが加算されるまでに距離に応じたタイムラグが発生する
- → 首都近傍は即時反映だがスペース限定、遠方は高産出だが遅延というトレードオフ

### 時間スケール

- 最小時間単位: **1 hexadies = 6日**
- 1ヶ月 = 5 hexadies = 30日
- 1年 = 12ヶ月 = 60 hexadies = 360日
- 内部表現は整数 hexadies（`i64`）
- 1ゲームの長さ: **Stellarisと同程度**（リアル時間）
- ゲーム内で数千年規模の文明史をプレイする

### 相対論的効果

- **時間の遅れ（time dilation）はゲームメカニクスとしては無視する**
- 全フレームが同一時刻を共有し、全エンティティの時間は同速で進む
- フレーバーテキストやイベントで言及する程度に留める

### FTL速度

- 初期段階で**10c**（光速の10倍）
- テクノロジーによりアップグレード可能
- 参考到達時間（初期）:
  - 4 ly先 → 約24 hexadies（≈5ヶ月）
  - 30 ly先 → 約180 hexadies（3年）
  - 100 ly先 → 600 hexadies（10年）

### 命令価値システム

- 遠隔地への命令には**命令価値（command value）**が付与される
- 有効命令価値 = 基礎命令価値 / (1 + α × 伝達時間)
- 伝達時間が長いほど命令価値が割引される
- 同一対象への複数命令がコンフリクトした場合、**有効命令価値が最大の命令のみが実行される**
- これにより、プレイヤーが後から送った命令（伝達時間が短い）が自然に優先される
- 通信手段によって伝達時間が異なるため、高速通信インフラの価値が高まる

### 経済モデル

- MVP段階の資源は3種: **鉱物**、**エネルギー**、**研究ポイント**
- 希少資源等は後続フェーズで追加予定

### プレイヤーの正体

- 抽象的な「**統治者**」として扱う
- 不老・永続的存在（数千年にわたってプレイ）
- 具体的なキャラクター設定は国家体制により異なる（MVP後に拡張）

### MVP スコープ

- **シングルプレイヤーのみ**（他国家はMVPから除外）
- 探索・入植・通信遅延の体験がMVPの核

### テクノロジーツリー

4ブランチ構成：

1. **社会学** — 人口成長、統治効率、自律AI性能、文化的影響力
2. **基礎物理学** — FTL速度UP、FTL射程UP、亜光速UP、測量範囲UP、通信技術
3. **産業技術** — 生産力UP、新建築物、建設速度、資源効率
4. **軍事技術** — 艦船HP、武装、防御、新艦種、早期警戒

- 各ブランチに研究ポイントコストを持つテクノロジーが連なる
- 前提条件あり（ツリー構造）
- 研究キューで1つずつ研究（Stellaris式）
- 研究完了時にグローバルパラメータを変更
- **遠隔地の研究ポイントは光速遅延後に首都に加算される**（既定）

### 探索イベント

測量完了時にランダムで発生：

- **資源ボーナス** — 星系の資源レベルUP（「豊富な鉱脈を発見」）
- **遺跡発見** — 研究ポイントボーナス or 特殊テクノロジー解放
- **危険** — 入植コスト増（「不安定な恒星」）、艦船ダメージ（「宇宙生物」）
- **特殊** — 追加の入植スロット（「居住可能な衛星」）

### 領有権システム

スコアベースの領有権：

- 各星系に対して各陣営の**支配権スコア**を計算
- **スコア最大の陣営**がその星系を領有（ただし閾値以上の場合のみ）
- 閾値未満 → 「未領有」（誰でも入植可能）
- 複数陣営が閾値以上で拮抗 → 「係争地」

支配権スコアの源泉：
- **コロニー人口** — 最大の影響力源
- **駐留艦船** — 軍事プレゼンス
- **通信可達性** — 定期的に命令が届いている度合い
- **距離減衰** — 首都/最寄り入植地からの距離で減衰

通信遅延との連携：
- 通信網が途絶すると通信可達性スコアが低下
- 遠方コロニーの支配権が揺らぎ、自律AIへの依存度が増す
- MVP段階ではシングルプレイヤーのため、コロニー有 = 領有として簡略化

### 入植フロー

- 入植船（ColonyShip）が星系に到着しても**自動入植しない**
- プレイヤーが星系の属性を確認した上で**明示的に入植コマンド（C キー）**を実行
- 入植船は到着後 Docked 状態で待機し、別の星系に再出発させることも可能
- 入植実行時に入植船は消費され、コロニーが設立される

### 惑星開発

- 各コロニーに**建設スロット**がある（SystemAttributes.max_building_slots で定義済み）
- スロットに**建築物**を建設して生産力を強化：
  - 鉱山（鉱物UP）、発電所（エネルギーUP）、研究施設（研究UP）
  - 造船所（艦船建造速度UP、新艦種解放）
  - 防衛施設（将来の戦闘用）
- **生産フォーカス**の切り替え（鉱物重視/エネルギー重視/バランス）
- 遠隔地の建設指示にも通信遅延が適用される

### 艦隊

- 複数の艦船を明示的に**艦隊**として編成できる
- Fleet component: 所属艦のリスト + 旗艦指定
- **艦隊の移動速度** = 所属艦の最低速度に合わせる
- **艦隊のFTL射程** = 所属艦の最小射程に合わせる
- 艦隊に命令 → 全艦が同時に移動、同時に到着
- 艦隊の解散は明示的に行う
- プレイヤーは艦隊単位で搭乗（旗艦を指定）
- 通信艦を艦隊に含めると、亜光速移動中も帝国と通信可能

### 戦闘システム

#### 基本方針
- 通信遅延により、プレイヤーが遠隔地の戦闘を直接操作できないことが多い
- **事前計画（交戦規定）+ 自動解決** がベース
- プレイヤー搭乗時のみ直接指揮が可能

#### 戦闘解決
- 同じ星系に敵対勢力と自陣営の艦船がいる場合に交戦
- **自動解決**: 双方の戦力値を比較し、ラウンドごとにダメージを計算
- 戦力値 = Σ(各艦の攻撃力) vs Σ(各艦の防御力 + HP)
- 敗勢側は交戦規定に基づいて撤退判断

#### 戦闘ステータス
- 各艦船に **attack**, **defense**, **hp/max_hp**
- 軍艦(Warship): 高い攻撃力/防御力、テクノロジーで解放
- 既存艦種の戦闘力は低い（Explorer/Courier は戦闘向きではない）

#### 交戦規定（ROE: Rules of Engagement）
- 自律AIに事前設定する戦闘方針
- **攻撃的**: 敵を見つけたら交戦
- **防衛的**: 攻撃された場合のみ応戦
- **撤退優先**: 敵がいたら撤退
- 交戦規定の変更にも通信遅延がかかる

#### プレイヤー搭乗時
- プレイヤーが搭乗中の艦隊は直接指揮可能（遅延なし）
- 撤退判断、ターゲット選択などをリアルタイムで行える
- **旗艦が撃沈された場合: プレイヤーは首都にリスポーン**
- 搭乗艦を失い、遠征先の直接管理ができなくなるペナルティ

#### 敵対的勢力（初期配置）
- 銀河生成時に一定割合(10-15%)の星系に配置
- 種類: 宇宙生物、古代の自動防衛機構 等
- 測量時に存在が判明
- 入植前に排除が必要
- 戦力はレベル制（弱い/中/強い）

### 維持費

- 艦船・建築物はエネルギーの維持費が毎hexadies発生
- 維持費が払えない場合: 生産停止や性能低下
- 拡張しすぎると維持費で首が回らなくなる制約

### リソースモデル

- **鉱物/エネルギー** — 完全ローカル（各コロニーの stockpile に蓄積）
- コロニー間の移動はクーリエによる物理輸送
- テクノロジーで自動交易ルートを研究可能
- **研究ポイント** — フロー型（蓄積不可、使わなければ消滅）
- 各コロニーで生産、光速遅延で首都に集約
- 研究開始時に鉱物/エネルギーは前払い一括消費

## 未確定事項（後続フェーズ）

- 外交システム（ただし通信遅延は適用する）
- 種族・帝国作成
- 勝利条件（他国家追加後に設計）
- 希少資源・交易
- 自然災害・危機イベント
- 自律AIの暴走・反乱メカニクス

### ループの特徴

1. **情報の非対称性** — プレイヤーは帝国全体を同時に把握できない。近くは鮮明、遠くは古い
2. **委任の必然性** — 帝国が広がるほど直接管理できる割合が減り、AIへの委任が不可避になる
3. **移動のコスト** — 現在位置を離れること自体がリスクとコストを伴う重大な意思決定
4. **通信インフラ = 国力** — 通信網の整備度が帝国の実効支配力を決定する
