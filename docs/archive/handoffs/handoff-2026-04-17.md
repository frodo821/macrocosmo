## 主要アーキテクチャ変更

### 1. ShipState::Docked 廃止 → InSystem + DockedAt

`ShipState::Docked { system }` は「星系にいる」と「harbour に dock している」を混同していた。

- `ShipState::InSystem { system }` — 星系に遊弋
- `DockedAt(Entity)` — orthogonal component、harbour ship に dock 中
- 組み合わせ: InSystem+DockedAt=停泊中、InSystem のみ=遊弋、DockedAt+InFTL=carrier 移動 (将来)

### 2. Capability HashMap 全廃 → modifier (ShipStats CachedValue)

`capabilities: HashMap<String, ...>` が hull/building/structure の 3 箇所に散在していた問題を解消。

- `harbour_capacity` は `ShipStats` の `CachedValue` (module modifier で加算)
- `harbour_capacity > 0` = harbourable (boolean flag 不要)
- station hulls は `size = 10000` で fleet capacity を超過 → 命令不可 (is_immobile() チェック不要化)

### 3. Modifier routing 拡張 — `context:filter::target` 統一構文

```
ship.*                           — 自分自身
system.*                         — 所在 system
fleet.*                          — 所属 fleet
docked_to:self::ship.*           — harbour module → docked ships
docked_to:<hull_id>::ship.*      — tech → 特定 harbour の docked ships
docked_to:*::ship.*              — empire → 全 docked ships
job:<job_id>::colony.*           — 既存
```

### 4. Hull intrinsic vs modifier の分離

| Hull intrinsic (不変) | Module modifier (可変) |
|---|---|
| `size` — 物理的大きさ | `ship.harbour_capacity` |
| `is_capital` — fleet 別枠判定 | `fleet.command_capacity` |
| slot 定義 | `system.sovereignty_claim` |

### 5. ROE-based combat for docked ships

- Aggressive/Defensive → hostile 出現で auto-undock → combat 参加
- Evasive/Passive → docked のまま (harbour が盾)、harbour 破壊で forced undock
- `UndockedForCombat(Entity)` marker で auto-return dock

### 6. DiplomaticOption framework (#302)

- `define_diplomatic_option` Lua API (旧 `define_diplomatic_action` は #325 で廃止)
- `DiplomaticEvent` component + `DiplomaticInbox` — light-speed delay 付き bilateral dispatch
- `Faction.allowed_diplomatic_options: HashSet<String>` (FactionType preset)
- POD responses (closure 禁止、serializable)

### 7. Casus Belli system (#305)

- `define_casus_belli` Lua API + `CasusBelliRegistry`
- `ActiveWar` / `ActiveWars` — single CB per war
- `evaluate_casus_belli` auto-war trigger system
- `end_war` + `available_end_scenarios`

## 確定した設計判断 (memory 更新推奨)

| DD | 決定 |
|---|---|
| Capital ship 判定 | hull flag `is_capital: bool`、size ベースではない |
| Fleet × capital | 初期は隠し fleet 固定、tech unlock `fleet.command_capacity` で編入可能 (別 issue) |
| Capital ship ≠ harbourable | 独立軸 (harbour なし capital もあり得る) |
| Resource sourcing | InSystem Shipyard → system stockpile、Loitering → own cargo + docked cargo |
| Nesting prohibition | size soft forbid (station size=10000 >> capacity)、engine hardcode 不要 |
| is_immobile() 廃止方向 | fleet capacity gate で統一 (size 超過 → 命令不可) |
| Capability 概念廃止 | ShipStats typed field (CachedValue) で表現、HashMap<String, ...> 全廃 |
| UI 大数値表示 | SI prefix (k/M/G/T/P) が望ましい (memory 保存済み) |
