### 3. mlua aux stack leak 解消 (release blocker #320)

**本セッション最大の発見**:

- 症状: 80 tick 以上で `evaluate_fire_conditions` が panic (`cannot create a Lua reference, out of auxiliary stack space`)
- 原因: `build_gamestate_table` が ~101 Lua refs / build を生成、`lua.gc_collect()` 未発火で Lua registry に蓄積、80 tick × 100 ref ≈ 8000 = `LUAI_MAXCSTACK` 限界
- Plan agent の実測で **leak source 2 箇所** 判明: `evaluate_fire_conditions` (issue 指摘) + `dispatch_event_handlers` (agent 発見)
- Fix: `gc_collect()` を両 closure 末尾に + Periodic fire_fn None filter (MTTH と対称に) + 1000 tick stress test
- regression guard: `HEAD~2` に revert して stress test が panic することを確認 (#320 正確症状を再現)
- follow-up: #328 per-tick gamestate cache (build 回数 1/tick に削減、visibility contract "mutation は次 tick") を起票

## 設計で確定したこと

### 1. save format = postcard (bincode 廃止)

- 2026-01 頃に bincode maintenance 終了のため **postcard v1** 採用済 (`Cargo.toml`)
- 既存 `src/persistence/` で `postcard::to_stdvec` / `from_bytes` 使用中
- memory: `project_save_format_postcard.md`

### 2. Lua event.gamestate = snapshot-per-event (pivot 済)

- #263 実装時に mlua 0.11 の scoped UserData は method 内から child UserData を返せない制約が判明、**live World view を断念して snapshot** に pivot
- 単一 callback 内で live view と観測上等価 (callback 中 world mutate しない)
- List shape は weak seal (LuaJIT `ipairs` 制約)、techs/flags は `__index(id) -> bool` のみ (`pairs()` 不可)
- **#328 で per-tick cache 化予定** — lazy build、"mutation は次 tick で反映" の visibility contract を document

### 3. HostilePresence 完全廃止 (root-cause ordering flip)

- `spawn_hostile_factions` を `.after(spawn_player_empire)` に (galaxy 依存削除)、`generate_galaxy.after(spawn_hostile_factions)` で直接 `FactionOwner` 付与
- `attach_hostile_faction_owners` backfill system 削除
- `AtSystem` / `HostileHitpoints` / `HostileStats` / `Hostile` marker に分解
- `FactionTypeRegistry.strength/evasion/default_hp/default_max_hp` を Lua で定義可能に

### 4. NPC AI = pluggable mock (実 AI は `macrocosmo-ai` 配下で)

- `trait NpcPolicy { fn tick(...) -> Option<Command> }` + `NoOpPolicy` default
- `macrocosmo-ai::mock` feature は dev-dep 限定 (本番 feature 露出なし)
- 本番 NPC decision tick は hand-written no-op、実 AI は `macrocosmo-ai::campaign/nash/feasibility` を wire する別 issue (#189 配下)

### 5. Gamestate cache visibility contract (#328 で実装)

> World mutations become visible in the **next tick's** gamestate snapshot. Within a single tick, all callbacks observe the same view regardless of when they fire. Systems that need to react "instantaneously" to a mutation must use direct ECS access, not `event.gamestate`.

光速通信遅延を core mechanic にしている本ゲームと思想整合。
